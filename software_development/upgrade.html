2. Upgrade¶

2.1. Upgrading from v1.7 to v1.8¶

Multiple versions of the SDK software may be installed on the same system, with the version currently in use reflected by the $GMT_GLOBAL environment variable.

  1. Upgrade NodeJS from version 10 (used on SDK < v1.7) to version 12.
$ sudo dnf remove -y nodejs
$ sudo dnf module reset -y nodejs
$ sudo dnf module enable -y nodejs:12
$ sudo dnf install -y nodejs
  1. If you are using the RT kernel, temporarily remove ssh from the list of update packages (see installation page for more info)
  1. Download the latest SDK distribution:
$ sudo wget http://52.52.46.32/srv/gmt/releases/sdk/linux/gmt-sdk.tar.gz
  1. Extract the TAR file in the /opt directory, into a new folder for the latest release:
$ sudo mkdir /opt/gmt_release_1.8.0
$ sudo tar -xzvf gmt-sdk.tar.gz -C /opt/gmt_release_1.8.0
  1. Update the symbolic link from the Global GMT Software Location to the latest release:
$ sudo ln -sfn gmt_release_1.8.0 /opt/gmt
  1. Check all applicable environment variables
$ gmt_env

To create the user working development environment, please follow the instructions on the installation page.

2.2. Upgrading from v1.5 to v1.6 and v1.7¶

Multiple versions of the SDK software may be installed on the same system, with the version currently in use reflected by the $GMT_GLOBAL environment variable.

  1. Download the latest SDK distribution:
$ sudo wget http://52.52.46.32/srv/gmt/releases/sdk/linux/gmt-sdk.tar.gz

for the server version of the SDK, or

$ sudo wget http://52.52.46.32/srv/gmt/releases/sdk/macos/gmt-ui.tar.gz

for the workstation version.

  1. Extract the TAR file in the /opt directory, into a new folder for the latest release:
$ sudo mkdir /opt/gmt_release_1.7.0
$ sudo tar -xzvf gmt-sdk.tar.gz -C /opt/gmt_release_1.7.0
  1. Update the symbolic link from the Global GMT Software Location to the latest release:
$ sudo ln -sfn gmt_release_1.7.0 /opt/gmt
  1. Check all applicable environment variables
$ gmt_env

To create the user working development environment, please follow the instructions on the installation page.

2.3. Upgrading from v1.5 to v1.6¶

Please follow the instructions on the SDK 1.6 migration guide page in order to migrate the code and the structure of the modules to the new formats.

2.4. Upgrading from v1.4 to v1.5¶

Because version 1.5 is built for Fedora 28, upgraded from Fedora 26, it is recommended to use the instructions on the Installing the SDK page to install on a clean system.

Multiple versions of the SDK software may be installed on the same system, with the version currently in use reflected by the $GMT_GLOBAL environment variable.

  1. Download the latest SDK distribution:
$ sudo wget http://52.52.46.32/srv/gmt/releases/sdk/linux/gmt-sdk-1.5.0.tar.gz

for the server version of the SDK, or

$ sudo wget http://52.52.46.32/srv/gmt/releases/sdk/macos/gmt-ui-1.5.0.tar.gz

for the workstation version.

  1. Extract the TAR file in the /opt directory, into a new folder for the latest release:
$ sudo mkdir /opt/gmt_release_1.5.0
$ sudo tar -xzvf <gmt-tar.gz> -C /opt/gmt_release_1.5.0

where <gmt-tar.gz> is the file downloaded in step 1.

  1. Update the symbolic link from the Global GMT Software Location to the latest release:
$ sudo ln -sfn gmt_release_1.5.0 /opt/gmt
  1. Check all applicable environment variables
$ gmt_env

These instructions assume that a working development environment has been set up before. To follow all steps for creating a new development environment, please follow the instructions on the Installing the SDK page.
